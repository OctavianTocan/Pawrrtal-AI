Next.js was overkill

No SEO. No SSR (Electron wraps a SPA). No edge runtime. No 'use server'. We were paying every Next.js cost for zero of its features.

⚡

Vite is the new shell

Same renderer Linear / VS Code / Discord / Slack / Notion / Cursor ship. 2026's convergent default. Dev server boots in ~550ms.

🧭

TanStack does routing

File-based routes (src/routes/*) replace app/*. proxy.ts middleware becomes a route beforeLoad auth probe.

The 5 things the PR actually does

Click each one to expand. They're independent — touch them in this order if you're auditing.

1

Swap the build tool

package.json · vite.config.ts · index.html ▸

The mechanical bits. next goes out, vite + plugins go in. Scripts get retargeted: dev → vite --port 3001, build → tsc && vite build, preview replaces start.

- "dev": "next dev --turbopack -p 3001"
+ "dev": "vite --port 3001"
- "build": "next build"
+ "build": "tsc && vite build"
- "start": "next start -p 3001"
+ "preview": "vite preview --port 3001"

frontend/index.html is now the HTML shell (replaces app/layout.tsx's <html>). The Google Fonts <link> URL is identical to what app/layout.tsx already had — fonts that were going through next/font/google are now in the same one-liner.

2

Move every route

app/ → src/routes/ ▸

1-to-1 file moves. The route tree is the same; the filenames change to match TanStack's convention.

Next.js (before)TanStack (after)

app/(app)/page.tsxsrc/routes/_app/index.tsx

app/(app)/dashboard/page.tsxsrc/routes/_app/dashboard.tsx
app/(app)/knowledge/page.tsxsrc/routes/_app/knowledge.tsx
app/(app)/tasks/page.tsxsrc/routes/_app/tasks.tsx

app/(auth)/signup/page.tsxsrc/routes/signup.tsx

Auth gate moved. proxy.ts middleware is gone — its job is now a TanStack beforeLoad on _app.tsx that probes the backend with a cheap authed call and redirects to /login on 401.

The conversation route gave up its server-side cookie fetch. c.$conversationId.tsx's loader now runs client-side with credentials: 'include'. Should still work, but worth eyeballing the first conversation load.

3

Strip directives + the navigation shim

172 files lose 'use client'; lib/navigation.ts added ▸

Vite has no client/server split, so every 'use client' / 'use server' just deletes. 172 files got swept.

Every from 'next/navigation' rewrites to a thin shim at frontend/lib/navigation.ts — it re-exports useRouter, usePathname, useSearchParams with Next-shaped signatures, but backed by TanStack Router. Call sites stay verbatim.

- import { useRouter } from 'next/navigation';
+ import { useRouter } from '@/lib/navigation';
// same .push() / .replace() shape; gradually migrate to native TanStack APIs.

Env var swap too: process.env.NEXT_PUBLIC_API_URL → import.meta.env.VITE_BACKEND_URL in lib/api.ts.

4

Teach Electron to serve a SPA

electron/src/server.ts ▸

Production prod path no longer spawns the Next.js standalone server. Now serves Vite's dist/ via a tiny built-in Node static server with SPA index.html fallback for client-side routes.

startNextServer kept as an alias so older code paths still work during the rebase window.

Stretch follow-up: the inline MIME map could swap for serve-static if that dep is allowed.

5

Patch tests + dev tooling

test/setup.ts · dev.ts ▸

Tests render outside <RouterProvider>. So test/setup.ts stubs vi.mock('@/lib/navigation', ...) with no-op router hooks instead of throwing.

Root dev.ts drops the Next dev-lock cleanup, keeps the workspace-filter spawn (Vite cares about port collisions just like Next did).
