5 landmines deliberately NOT touched

These look like they should be renamed. They aren't, because they're wire-protocol or persistent-state identifiers. Renaming them without a migration would break existing installs and stored data.

nexus:personalization

localStorage key on the frontend. Renaming it would silently wipe every existing user's personalization on first load.

nexus.db

SQLite filename in backend default config. Renaming detaches existing self-hosted installs from their database.

x-nexus-surface

HTTP header used by the Telegram channel router. Renaming desyncs anything in flight or cached upstream.

nexus_user_id

Internal Python identifier — appears as a DB foreign key + handler arg. Renaming = schema migration territory.

appId: "ai.nexus.desktop"

In electron/package.json. Changing this makes installers treat Pawrrtal as a brand-new app instead of an upgrade — users lose continuity, settings, app data.

→ Follow-up bean owed: schema rename, electron update channel migration, localStorage one-time copy-and-delete shim on first run.

License: MIT → FSL-1.1-Apache-2.0

Source-available now, fully open later. Strict enough to deter clones, lenient enough to attract contributors and self-hosters.

retired

MIT

Maximally permissive. Anyone can fork it tomorrow, slap a logo on it, and ship a paid hosted competitor. Fine for a hobby project; a problem if this becomes a commercial product.

in

FSL-1.1-Apache-2.0

Source-available. Blocks competing use for 2 years. Each release auto-converts to Apache-2.0 after that window. One-page license. Used by Sentry, Keygen — becoming the default for "open-ish but might commercialise".

// considered alternatives:
// - BSL-1.1: 4-page license, ~4y delay. Heavier paperwork, longer lockup. Skip unless you want the strictness.
// - GPL/AGPL: bans commercial proprietary forks but scares contributors. Wrong tradeoff for a small product.
// - MIT (current): too permissive once revenue is in play.

Cat theme, wired but not painted

PRODUCT_THEME = "cat" is in. No palette tokens are renamed in this PR — that's a follow-up so the rename stays mechanical. Future themes plug in beside cat: rotate the accent palette + icon set keyed on the theme constant.

What you do after this lands

4 steps. The repo rename is the only one that matters for outside-world URLs.

  1. Land this PR.
  2. Rename the GitHub repo: OctavianTocan/ai-nexus → OctavianTocan/pawrrtal (Settings → rename). GitHub auto-redirects the old URL, so workflows that reference the repo by URL keep working through the cutover.
  3. Rotate the local clone's remote: git remote set-url origin <new>
  4. Optional: pnpm install + uv sync to refresh lockfiles under the new package names.
